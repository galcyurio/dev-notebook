<servlet-mapping>
	<servlet-name>dispatcherServlet</servlet-name>
	<url-pattern>/*</url-pattern>
</servlet-mapping>

위와 같이 처음에 /* 로 mapping 할 경우
경고: No mapping found for HTTP request with URI [/WEB-INF/jsp/board/list.jsp] in DispatcherServlet with name 'dispatcherServlet'
와 같이 jsp 를 읽어들이지 못하게 된다.
(jsp 를 담당하는 맵핑에 걸리지 못하고 dispatcherServlet 에 걸려서...)

해결방법)
1. Default Servlet Handler 로 정적자원들을 forward 하기

<mvc:default-servlet-handler> 를 사용하면 static resources를 servlet container 의 default Servlet 에게로 forward 한다.
이를 통해 '/' 로 DispatcherServlet 을 mapping 할 수 있다.
이 handler 는 모든 요청들을 defaultServlet 으로 forward 하므로 모든 다른 UrlHandlerMapping 들을 마지막에 남겨둔다.
이렇게 다른 HandlerMapping 들이 여러가지가 남아있게 되면 이상하게 SiimpleUrlHandlerMapping 이 작동해버린다.(?)

여기에 <mvc:annotation-driven> 을 사용하여 DefaultAnnotationHandlerMapping 을 사용하도록 하면 정상적으로 작동한다.
